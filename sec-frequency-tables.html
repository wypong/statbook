    Section 2.1 Frequency Tables and Relative Frequency

    A frequency table organizes data by listing each value or category together with its frequency, which is the number of times that value occurs. If we divide each frequency by the total number of observations, we get the relative frequency. Relative frequency tells us the share of the data in each category.
    🔗
    \begin{equation*} \text{relative frequency} = \frac{\text{frequency}}{\text{total number of observations}} \end{equation*}
    Relative frequency is often written as a decimal, but a percentage is usually easier to read. The relative frequencies in a complete table should add up to \(1\text{,}\) or equivalently to \(100\%\text{.}\)
    🔗
    A quick way to practice these ideas is with data from the class itself. The point is not to make the numbers fancy. The point is to get comfortable turning a raw list into a table, then reading that table a little more carefully.
    🔗

    Activity 2.1.1. Last-Name Initials.

    Use the first letter of each student’s last name to build a frequency table. This is a nice warm-up because the data are categorical, but the table still has enough structure to let us talk about relative and cumulative frequency.
    🔗

    (a)

    Collect the first initial of each student’s last name. Make a frequency table with the initials in alphabetical order.
    🔗
    🔗

    (b)

    Add a relative frequency column. Which initial has the largest relative frequency? What percentage of the class does it represent?
    🔗
    🔗

    (c)

    Add a cumulative frequency column, again using alphabetical order. Which initials account for about half of the class?

    Activity 2.1.2. Length of Student Names.

    Now use the lengths of students’ names. To keep the counting consistent, count letters only and ignore spaces and punctuation. This gives us numerical data, so the table can also lead naturally into cumulative frequency.
    🔗

    (a)

    Count the number of letters in each student’s full name. Make a frequency table for the name lengths, ordering the lengths from smallest to largest.
    🔗
    🔗

    (b)

    Add a relative frequency column. Which name length is most common? Which lengths are rare?
    🔗
    🔗

    (c)

    Add a cumulative frequency column. As you move from the shortest names to the longest, at what name length does the cumulative frequency first pass half of the class? What proportion of the class do the shorter names represent?

    🔗
    Sometimes a variable has many possible values, or it is measured on a continuous scale. In that situation a row for every single value would not be very useful. Instead we group nearby values into class intervals. A table built from intervals is called a grouped frequency table.
    🔗
